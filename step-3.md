# Step 3 - 统一类型与标准化映射层

## 目标
建立项目统一类型、抓取结果类型和标准化映射逻辑，确保不同来源的数据在入库前都转换为同一结构。

## 输入
- [design.md](C:\Users\admin\Documents\yangjingjing\design.md) 中的统一字段定义
- Step 2 已稳定的数据模型
- 目标来源的字段样例或页面字段观察结果

## 输出
- 统一项目 TypeScript 类型
- 抓取原始数据类型定义
- 标准化后的中间结构类型
- `normalize` 或等价映射模块
- 至少 2 组标准化样例输入与输出

## 不做内容
- 不访问真实来源进行批量抓取
- 不写入数据库
- 不实现 API 接口
- 不实现页面交互

## 实现要求
- 统一类型需覆盖 `projects` 表的主要业务字段。
- 抓取类型与标准化类型分层定义，不能混用原始字段名和标准字段名。
- 标准化映射需处理：
  - 空值兼容
  - 字符串清洗
  - 年份转换
  - 来源标识统一
  - 原始链接保留
- 映射层只负责“转结构”，不直接负责数据库写入和页面展示。
- 保留 `sourceRaw` 所需的原始字段快照能力，便于排查抓取错误。
- 至少准备 2 组不同结构的样例数据，用于验证标准化规则，而不是只依赖口头说明。

## 校验方式
- 用至少 2 组模拟来源样例测试标准化结果。
- 缺失部分字段时，仍能得到合法的标准结构对象。
- 映射结果字段名与数据库模型一致。
- TypeScript 类型检查通过。
- 同一输入多次执行标准化时，输出结果保持稳定一致。

## 完成标准
- 后续每增加一个来源，只需要增加来源适配代码，不需要改项目主模型。
- 抓取层、入库层、接口层均可复用同一套标准类型。
- 标准化规则已足够清晰，可作为后续代码评审依据。
- 开发者拿到任意来源样例后，能明确判断它该如何映射到统一结构。
